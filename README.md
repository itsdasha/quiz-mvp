# QuizRoom MVP

Рабочий прототип веб-приложения для проведения квизов в реальном времени. Организатор создаёт квиз, запускает комнату и показывает вопросы, а участники подключаются по коду комнаты и отвечают только во время демонстрации текущего задания.

## Возможности

- регистрация и авторизация участников и организаторов;
- личный кабинет с историей участий и проведённых квизов;
- создание и редактирование квизов: категории, правила, время на вопрос;
- вопросы двух типов: текстовые и с изображением;
- одиночный и множественный выбор ответа;
- запуск комнаты и подключение участников по 6-значному коду;
- синхронное обновление экрана у всех участников через WebSocket;
- приём ответов только во время показа активного вопроса;
- автоматический подсчёт баллов и итоговый лидерборд.

## Технологии

- **Клиент:** HTML, CSS, JavaScript (одностраничное приложение);
- **Сервер:** Node.js — HTTP API и WebSocket без внешних зависимостей;
- **Хранилище:** JSON-файл `data/db.json` (создаётся автоматически при первом запуске; для промышленной версии заменяется на PostgreSQL или MongoDB).

## Запуск

Требуется [Node.js](https://nodejs.org/) версии 18 или новее.

```bash
npm start
```

Приложение откроется на `http://localhost:3000`.

При первом запуске автоматически создаются демо-аккаунты и демо-квиз:

```text
Организатор: organizer@example.com / demo1234
Участник:    participant@example.com / demo1234
```

## Как проверить сценарий квиза

1. Войдите как организатор.
2. Откройте демо-квиз или создайте новый.
3. Нажмите «Запустить» и скопируйте код комнаты.
4. В приватном окне браузера или на другом устройстве войдите как участник.
5. Введите код комнаты.
6. Организатор нажимает «Показать первый вопрос».
7. Участник отвечает до окончания таймера.
8. После всех вопросов все видят итоговый лидерборд, результат сохраняется в истории.

## Структура проекта

```text
quiz-mvp/
  docs/

  public/
    index.html          # HTML-страница приложения
    styles.css          # стили интерфейса

  server/
    server.js           # Node.js сервер: статика, HTTP API, WebSocket
  package.json
```

## Архитектура

Клиент обращается к HTTP API для регистрации, входа, создания квизов, запуска комнат и отправки ответов. Все изменения состояния комнаты — подключение участников, новый вопрос, ответы, завершение — сервер рассылает подключённым клиентам по WebSocket, поэтому экран обновляется у всех одновременно.

Правильные ответы никогда не отправляются участникам до завершения вопроса, ответы принимаются только в отведённое время, повторные ответы отклоняются.
